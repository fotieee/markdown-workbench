# Security Policy

## Supported Versions

Security fixes are handled on the latest released version.

## Reporting a Vulnerability

Please do not publish security issues publicly before they are reviewed.

If you find a vulnerability, open a private report through the repository security page after the project is published, or contact the maintainer directly.

## Notes

Markdown Workbench reads local files selected by the user. Renderer access to Node.js is disabled, and privileged file operations are exposed through a small preload API.
